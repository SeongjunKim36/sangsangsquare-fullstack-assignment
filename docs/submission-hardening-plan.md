# 제출 전 필수 개선 계획

## 목표

- 채용 과제 제출물의 신뢰도를 높인다.
- 새로운 기능을 추가하지 않고, 현재 설계를 끝까지 검증 가능한 상태로 마감한다.
- 오버엔지니어링 없이 필수 위험만 제거한다.

## 진행 상태

- [x] 테스트 DB 분리
- [x] 동시성 E2E 안정화
- [x] 인증 수명주기 정리
- [x] trim 기반 입력 검증
- [x] 문서와 계정 정보 정합성 정리

## 범위

### 이번에 반드시 정리할 것

1. E2E 테스트가 제출용 SQLite를 직접 건드리지 않도록 테스트 DB를 분리한다.
2. 동시성 E2E가 실제로 통과하도록 테스트 실행 방식을 안정화한다.
3. 인증 수명주기를 닫는다.
4. 서버 입력값을 trim 기반으로 검증해 공백 문자열을 막는다.
5. README와 보조 문서를 현재 코드/계정/검증 결과와 맞춘다.

### 이번에 하지 않을 것

- SQLite를 PostgreSQL로 교체
- JWT, Redis Session Store, WebSocket 도입
- CQRS, 별도 Query Layer, 캐시 레이어 추가
- OpenAPI codegen, 런타임 스키마 시스템 도입

## 작업 계획

### 1. 테스트 DB 분리

#### 문제

- E2E 테스트가 제출용 `assignment.sqlite`를 직접 사용한다.
- 테스트 실행만으로 샘플 데이터와 스키마 상태가 바뀔 수 있다.

#### 변경

- TypeORM 설정을 환경별로 나누고, 테스트에서는 별도 SQLite 파일을 사용한다.
- 테스트 전후로 전용 DB 파일과 WAL/SHM 파일을 정리한다.
- 테스트 환경에서는 과도한 SQL logging을 끈다.

#### 완료 기준

- E2E 실행 후 `apps/server/data/assignment.sqlite`가 테스트 때문에 오염되지 않는다.

### 2. 동시성 E2E 안정화

#### 문제

- 현재 E2E는 병렬 PATCH 시나리오에서 실패하거나 환경 의존적으로 흔들린다.
- 과제의 핵심 강점인 정원 초과 방지 검증이 녹색으로 남아 있지 않다.

#### 변경

- Supertest 대상을 Express 앱 인스턴스가 아니라 Nest HTTP 서버로 통일한다.
- 로그인/쿠키 획득도 동일한 서버 객체 기준으로 수행한다.
- 테스트가 실제로 검증하려는 결과가 비즈니스 응답으로 드러나도록 조정한다.

#### 완료 기준

- `pnpm --filter server exec jest --config ./test/jest-e2e.json --watchman=false --runInBand`가 통과한다.

### 3. 인증 수명주기 정리

#### 문제

- 로그아웃이 세션 객체를 완전히 종료하지 않는다.
- 프론트의 인증 캐시가 너무 오래 살아 UI가 잠시 오래된 로그인 상태를 믿을 수 있다.

#### 변경

- 서버 로그아웃에서 세션을 destroy 하고 세션 쿠키를 정리한다.
- 프론트의 현재 사용자 조회는 항상 최신 세션 상태를 더 빠르게 반영하도록 조정한다.

#### 완료 기준

- 로그아웃 직후 보호 페이지 접근 시 인증 상태가 즉시 반영된다.

### 4. trim 기반 입력 검증

#### 문제

- 프론트에서 trim을 하더라도 서버가 공백 문자열을 최종적으로 막지 못한다.

#### 변경

- 로그인 DTO와 모임 생성 DTO에서 문자열을 trim 한 뒤 검증한다.
- 제목과 아이디처럼 비어 있으면 안 되는 값은 공백만 입력해도 실패하게 만든다.

#### 완료 기준

- `"   "` 같은 값이 서버 DTO 검증을 통과하지 못한다.

### 5. 문서와 계정 정보 정합성 정리

#### 문제

- README, 서버 README, 현재 아키텍처 문서의 계정/검증 설명이 서로 어긋난다.

#### 변경

- 실제 seed 계정 기준으로 문서를 단일화한다.
- README의 구현 설명은 현재 코드와 맞는 수준까지만 남긴다.
- 실행/검증 명령은 실제 통과한 명령만 남긴다.

#### 완료 기준

- 리뷰어가 README와 문서만 보고 실행해도 코드와 다른 설명을 만나지 않는다.

## 검증 명령

```bash
pnpm --filter web lint
pnpm --filter web build
pnpm --filter server lint
pnpm --filter server build
pnpm --filter server exec jest --config ./test/jest-e2e.json --watchman=false --runInBand
```
